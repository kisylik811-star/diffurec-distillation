# Consistency Distillation for DiffuRec

This module adds consistency-distillation training on top of DiffuRec, following
Song et al., "Consistency Models" (ICML 2023).

## Files

- `src/consistency_diffurec.py` — `ConsistencyDiffuRec` (student diffusion module)
  and `ConsistencyStudent` (full student model).
- `src/distill_trainer.py` — Distillation training loop and evaluation utilities.
- `src/distill_main.py` — Entry point: trains or loads a teacher, then distills
  a student and evaluates at NFE = 1, 2, 4, 8.

## Approach

The teacher is a regular DiffuRec doing T = `diffusion_steps` reverse passes per
prediction (32 by default). The student is initialized from the teacher and
trained so that a single forward pass `f_theta(x_t, t, history)` produces a good
x_0 for any t. The consistency target is generated by a deterministic DDIM step
from the frozen teacher.

The total loss is:

```
loss = cons_weight * MSE( f_theta(x_high, t_high), f_theta_ema(x_low, t_low) )
     + ce_weight   * CE  ( scores( f_theta(x_high) ), target )
```

The CE term keeps the student grounded in the recommendation task.

## Running (Colab)

From `DiffuRec/src/`, train teacher + student in one shot:

```bash
python distill_main.py --dataset amazon_beauty --epochs 200 --distill_epochs 100
```

Subsequent runs can skip teacher training:

```bash
python distill_main.py \
    --dataset amazon_beauty \
    --teacher_ckpt checkpoints/teacher.pt \
    --distill_epochs 200
```

## Output

The script logs:

- Teacher metrics at NFE = T (full reverse loop) and its latency.
- Per-epoch consistency loss + CE loss during distillation.
- Validation HR/NDCG at NFE = 1.
- Final test HR/NDCG at NFE = 1, 2, 4, 8.
- Inference latency at each NFE.

## Known issue in the upstream repo

`src/step_sample.py` contains:

```python
self._loss_counts = np.zeros([self.num_timesteps], dtype=np.int)
```

`np.int` is removed in NumPy >= 1.24, which is what current Colab uses.
Fix by replacing with `dtype=np.int64`.
